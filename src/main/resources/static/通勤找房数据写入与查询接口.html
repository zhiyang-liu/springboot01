1 概述

为了方便爬虫端数据的处理与C端查询的请求特编写此文档。通勤找房数据写入与查询主要包括针对通勤找房的3个必要接口。第一个接口提供向redis写入爬虫获得的通勤数据，第二个接口提供从楼盘字典写入小区信息至redis，第三个接口提供根据城市id、查询点的经纬度、通勤方式、通勤时间等检索出满足条件的小区信息以供C端使用。

1.1 Api签名

1.1.1 签名参数共包括3个：signCode、appId和mt

signCode：将除了 signcode 之外的其他参数按照 key 的字母序进行排序。按照key1,value1#key2,value2的顺序进行拼接字符串，然后对字符串进行sha256Hex编码。

appId：为C端请求的appId，为每一个C端应用分配唯一的一个appId，如果想使用通勤找房服务需要申请appId。

mt：时间戳，与现在的时间不能相差超过30分钟，该字段应该及时修正，已保证请求时间不超过30分钟。

例如：

1.1.2 签名 Java Demo

 
xxxxxxxxxx
public static String sign(HttpServletRequest request, String appkey) {
    List<String> keys = new ArrayList<>();
    Enumeration<String> paramNames = request.getParameterNames();
    while (paramNames.hasMoreElements()) {
        String pn = paramNames.nextElement();
        if (pn.equals("signCode")) continue;
        keys.add(pn);
    }
    Collections.sort(keys);
    StringBuilder sb = new StringBuilder();
    keys.forEach(k -> sb.append(k).append(",").append(request.getParameter(k)).append("#"));
    LOGGER.info("[未加密原值：{}]", sb.toString());
    sb.append(appkey);
    return DigestUtils.sha256Hex(sb.toString());
}

3 数据查询接口

数据查询服务主要向链家网有通勤找房需求的应用提供服务，根据城市id、查询点的经纬度、通勤方式、通勤时间等检索出满足条件的小区信息，再根据满足条件的小区自己从楼盘字典获取房屋信息，就完成了通勤找房功能。

  • 接口描述：查询满足通勤要求的小区信息
  • 请求url：/getResblocks
  • 请求方式：post
  • 参数：

commuteType：枚举类型，现共包括四种出行方式：WARKING、RIDE_BIKE、TAKE_BUS、DRIVING。
